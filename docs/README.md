## Documentation Standards

When contributing documentation:

1. **Keep it up to date**: Update docs when making code changes
2. **Be specific**: Include code examples and exact commands
3. **Use clear structure**: Headers, lists, and code blocks
4. **Link related docs**: Cross-reference between documents
5. **Include troubleshooting**: Document common issues and solutions
